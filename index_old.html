a-entity and components
Up until now we've been maily using primitives like a-box and a-sphere etc... There is a more versatile way of creating objects using the a-entity tag. Entities are placeholder objects to which we plug in components to provide them appearance, behavior, and functionality.
By itself the a-entity tag is just a place holder. We can attach components to it to make it render something or do something. To give it shape and appearance, we can attach the geometry, material, light, animation and other components. Let's look at ent1.html
  • Parent and Child Transforms
  • Entities can have a single parent and multiple children.
  • Child entities inherit transformations (i.e., position, rotation, and scale) from their parent.
  • You can also group things in an a-entity tag

Components of A-Frame’s entity-component framework are JavaScript modules that can be mixed, matched, and composed onto entities to build appearance, behavior, and functionality. We can register component in JavaScript and and add it to an a-entity tag. Components are configurable, reusable, and shareable. Most code in an A-Frame application should live within components.
